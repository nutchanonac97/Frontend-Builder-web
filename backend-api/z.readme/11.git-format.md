# คู่มือ Git Commit Convention 📝

## รูปแบบพื้นฐาน
```
<ประเภท>(<ขอบเขต>): <คำอธิบาย>

[รายละเอียดเพิ่มเติม (ถ้ามี)]

[หมายเหตุท้าย (ถ้ามี)]
```

## ประเภท (Types) หลักที่ใช้บ่อย 🔴

### ฟีเจอร์และบัค
- **feat**: เพิ่มฟีเจอร์ใหม่
- **fix**: แก้ไขบัค

### คุณภาพโค้ด
- **refactor**: ปรับปรุงโค้ดโดยไม่เปลี่ยนการทำงาน
- **perf**: ปรับปรุงประสิทธิภาพ
- **style**: แก้ format, ช่องว่าง, semicolon (ไม่กระทบ logic)

### การทดสอบและเอกสาร
- **test**: เพิ่ม/แก้ไข test cases
- **docs**: แก้ไขเอกสาร README, คอมเมนต์

### การบำรุงรักษา
- **build**: แก้ไขระบบ build, dependencies
- **ci**: แก้ไขการตั้งค่า CI/CD
- **chore**: งานบำรุงรักษาทั่วไป, อัพเดท packages
- **revert**: ยกเลิก commit ก่อนหน้า

## ตัวอย่างการใช้งาน 💡

### ฟีเจอร์ใหม่
```bash
feat(auth): เพิ่มการล็อกอิน OAuth2 ด้วย Google
feat(payment): พัฒนาระบบชำระเงินผ่าน Stripe
feat(api): เพิ่ม pagination ให้ endpoint รายการสินค้า
```

### แก้บัค
```bash
fix(cart): แก้ปัญหาการอัพเดทจำนวนสินค้าไม่บันทึก
fix(ui): แก้การจัดตำแหน่งปุ่มบนมือถือ
fix(validation): จัดการกรณีอีเมลเป็น null
```

### ปรับปรุงโค้ดและประสิทธิภาพ
```bash
refactor(utils): ทำให้ฟังก์ชันจัดรูปแบบวันที่ง่ายขึ้น
perf(database): เพิ่ม index เพื่อเร่งความเร็ว query
perf(api): ใช้ระบบ cache สำหรับข้อมูลที่เรียกใช้บ่อย
```

### การทดสอบ
```bash
test(auth): เพิ่ม unit tests สำหรับ service การล็อกอิน
test(e2e): เพิ่ม integration test สำหรับขั้นตอนการสั่งซื้อ
```

### เอกสาร
```bash
docs(readme): อัพเดทวิธีการติดตั้ง
docs(api): เพิ่มเอกสาร swagger สำหรับ endpoints
```

## แนวทางการระบุขอบเขต (Scope) 🎯

ขอบเขตระบุส่วนที่ถูกแก้ไข เช่น:
- **คอมโพเนนต์**: `(header)`, `(footer)`, `(sidebar)`
- **โมดูล**: `(auth)`, `(payment)`, `(user)`
- **ฟีเจอร์**: `(login)`, `(checkout)`, `(search)`
- **เลเยอร์**: `(api)`, `(database)`, `(ui)`

## แนวปฏิบัติที่ดี ✅

### ควรทำ
1. **ใช้ภาษาอังกฤษหรือไทย**: เลือกภาษาเดียวให้สม่ำเสมอ
2. **ตัวอักษรพิมพ์เล็ก**: ยกเว้นชื่อเฉพาะ
3. **กระชับชัดเจน**: ไม่เกิน 50 ตัวอักษรในคำอธิบาย
4. **เจาะจง**: บอกว่าทำอะไร ไม่ใช่ทำไม

### ไม่ควรทำ
1. ❌ `แก้บัค` - ไม่บอกว่าบัคอะไร
2. ❌ `อัพเดทโค้ด` - กว้างเกินไป
3. ❌ `WIP` - ควร commit เมื่อทำเสร็จ
4. ❌ ใช้ emoji ในคำอธิบาย (ยกเว้นทีมตกลง)

## การเปลี่ยนแปลงที่กระทบเวอร์ชันเก่า ⚠️

ถ้ามีการเปลี่ยนแปลงที่กระทบ backward compatibility:
```bash
feat(api)!: เปลี่ยนโครงสร้าง response ของ endpoint ผู้ใช้

BREAKING CHANGE: เปลี่ยนโครงสร้าง response จาก array เป็น object
```

## Commit แบบหลายบรรทัด 📄

สำหรับ commit ที่ซับซ้อน:
```bash
feat(checkout): พัฒนาระบบชำระเงินแบบหลายขั้นตอน

- เพิ่มฟอร์มที่อยู่จัดส่ง
- เพิ่มการเลือกวิธีชำระเงิน
- เพิ่มขั้นตอนตรวจสอบคำสั่งซื้อ
- เชื่อมต่อกับระบบชำระเงิน

ปิด #123
```

## ตารางอ้างอิงด่วน 📊

| ประเภท | ใช้เมื่อไหร่ | ตัวอย่าง |
|--------|-------------|----------|
| feat | ฟีเจอร์ใหม่ | `feat(cart): เพิ่มฟีเจอร์รายการโปรด` |
| fix | แก้บัค | `fix(login): แก้ปัญหา token หมดอายุ` |
| docs | เอกสาร | `docs(api): อัพเดทเอกสาร endpoint` |
| style | จัด format โค้ด | `style(global): ใช้ prettier จัดรูปแบบ` |
| refactor | ปรับปรุงโค้ด | `refactor(service): แยก logic การ validate` |
| test | เพิ่มการทดสอบ | `test(user): เพิ่มการทดสอบการสมัครสมาชิก` |
| chore | บำรุงรักษา | `chore(deps): อัพเดท dependencies` |

## Git Aliases สำหรับ Commit Types 🚀

เพิ่มใน `.gitconfig`:
```bash
[alias]
    # Commit ด่วนภาษาไทย
    ฟีเจอร์ = "!f() { git commit -m \"feat: $1\"; }; f"
    แก้บัค = "!f() { git commit -m \"fix: $1\"; }; f"
    เอกสาร = "!f() { git commit -m \"docs: $1\"; }; f"
    ปรับปรุง = "!f() { git commit -m \"refactor: $1\"; }; f"
    ทดสอบ = "!f() { git commit -m \"test: $1\"; }; f"
    
    # Commit ด่วนภาษาอังกฤษ
    cf = "!f() { git commit -m \"feat: $1\"; }; f"
    cx = "!f() { git commit -m \"fix: $1\"; }; f"
    cd = "!f() { git commit -m \"docs: $1\"; }; f"
    cr = "!f() { git commit -m \"refactor: $1\"; }; f"
    ct = "!f() { git commit -m \"test: $1\"; }; f"
    
    # พร้อมขอบเขต
    cfs = "!f() { git commit -m \"feat($1): $2\"; }; f"
    cxs = "!f() { git commit -m \"fix($1): $2\"; }; f"
```

วิธีใช้:
```bash
git ฟีเจอร์ "เพิ่มหน้าโปรไฟล์ผู้ใช้"     # feat: เพิ่มหน้าโปรไฟล์ผู้ใช้
git cf "add user profile page"          # feat: add user profile page
git cfs auth "เพิ่มระบบรีเซ็ตรหัสผ่าน"   # feat(auth): เพิ่มระบบรีเซ็ตรหัสผ่าน
```

## ตัวอย่างสำหรับโปรเจค Backend API 🔧

```bash
# Controller และ Endpoints
feat(api): เพิ่ม CRUD endpoints สำหรับสินค้า
feat(controller): พัฒนา endpoint อัพโหลดไฟล์
fix(api): แก้ status code สำหรับ validation errors

# ฐานข้อมูล
feat(db): เพิ่ม migration สำหรับโปรไฟล์ผู้ใช้
fix(db): แก้ปัญหา connection pool timeout
perf(db): เพิ่มประสิทธิภาพ query ด้วย indexing

# ระบบยืนยันตัวตน
feat(auth): พัฒนาระบบ JWT refresh token
fix(auth): จัดการ token หมดอายุอย่างถูกต้อง
test(auth): เพิ่มการทดสอบสำหรับการเข้าถึงตามบทบาท

# การตรวจสอบข้อมูล
feat(validation): เพิ่มตัวตรวจสอบอีเมลแบบกำหนดเอง
fix(validation): อนุญาตให้ใช้อักขระพิเศษในชื่อ

# เอกสาร
docs(swagger): เพิ่มตัวอย่าง request/response
docs(readme): อัพเดทขั้นตอนการ deploy
```

## เคล็ดลับการทำงานเป็นทีม 👥

1. **ตกลงรูปแบบในทีม**: เลือกประเภทที่จะใช้ร่วมกัน
2. **ใช้เครื่องมือช่วย**: commitizen, commitlint
3. **ตรวจสอบ commit messages**: ใน PR review
4. **สร้าง template**: ไฟล์ `.gitmessage`
5. **ทำให้เป็นอัตโนมัติ**: ใช้ pre-commit hooks

---

💡 **เคล็ดลับโปร**: ใช้ AI ช่วยสร้าง commit message:
```bash
# ใน Claude Code
git diff --staged | claude "สร้าง commit message ตาม conventional commits ภาษาไทย"
```

## ตัวอย่าง Commit Message ภาษาไทย 🇹🇭

```bash
# ฟีเจอร์
feat: เพิ่มระบบแจ้งเตือนผ่านอีเมล
feat(payment): รองรับการชำระเงินผ่าน PromptPay
feat(api): เพิ่ม rate limiting ป้องกันการใช้งานเกิน

# แก้บัค
fix: แก้ปัญหาการแสดงผลภาษาไทยไม่ถูกต้อง
fix(database): แก้การเชื่อมต่อ database หลุดบ่อย
fix(auth): แก้ปัญหาผู้ใช้ล็อกอินซ้ำ

# ปรับปรุง
refactor: จัดโครงสร้างโฟลเดอร์ใหม่ตาม clean architecture
perf: เพิ่มความเร็วการโหลดหน้าแรก 50%
style: จัดรูปแบบโค้ดตามมาตรฐาน PSR-12

# อื่นๆ
docs: เพิ่มคู่มือการใช้งาน API
test: เพิ่ม unit test ครอบคลุม 80%
build: อัพเกรด Node.js เป็นเวอร์ชัน 20
chore: ลบไฟล์ที่ไม่ใช้แล้ว
```
